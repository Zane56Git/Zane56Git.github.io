发表于 2019-03-19 更新于 2023-06-15 分类于 大数据 Changyan：

应用场景

分析统计，实时数据处理，图计算及机器学习

组件

  • Spark Core
  • Spark SQL
  • Spark Streaming
  • MLlib
  • GraphX

Spark开发

  • scala
  • java

环境搭建

Spark的运行模式

  • 1.local: 本地单进程模式，用于本地开发测试Spark代码。
  • 2.standalone:分布式集群模式，Master-Worker架构，Master负责调度，Worker负责具体Task的执行。
  • 3.on yarn/mesos:运行在yarn/mesos等资源管理框架之上，yarn/mesos提供资源管理，spark提供计算调度，并可与其他计算框架(如MapReduce/MPI/Storm)共同运行在同一个集群之上。
  • 4.on cloud(EC2): 运行在AWS的EC2之上

安装

Spark会用到HDFS与YARN，因此请先安装Hadoop

**第一步:**安装Java JDK 1.7及以上版本，并配置好环境变量。本电脑安装的jdk是1.7.0_79版本的。

**第二步:**安装Hadoop。本电脑安装的Hadoop是2.7.3版本的。

感谢&参考：https://www.cnblogs.com/yhl-yh/p/7651470.html

问题点

  • 解决A master URL must be set in your configuration错误

https://blog.csdn.net/shenlanzifa/article/details/42679577

  • master执行方式需要设置，可以再idea设置VM参数，也可以在代码中设置

-Dspark.master=local

  • ClassNotFoundException: com.mysql.jdbc.Driver

    解决方式是在idea中引入mysql-connector-java-5.1.47.jar（maven引入无效，很奇怪）

  • maven无法引入spark相关jar包，代码爆红

    很可能是因为spark和scala版本不兼容，可查看spark可对应的scala版本，调整即可

    例如从依赖的jar名中也能看出：spark-sql_2.11，2.11代表scala的版本

  • Error:(3, 25) object graphx is not a member of package org.apache.spark

    右键工程-open module setting-Libraries-spark-assembly-1.0.0-hadoop1.0.4.jar（添加spark的jar包-保存
