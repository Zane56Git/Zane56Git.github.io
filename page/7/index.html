Linux定时任务crontab实践-发送钉钉群消息

发表于 2020-05-17 更新于 2023-06-15 分类于 Linux Changyan：

感谢&参考：https://blog.csdn.net/chenyixuan0923_yp/article/details/100536890

1.、新建钉钉机器人，复制webhook地址

2、撰写shell脚本

  1. cd到/usr/local目录下，新建work目录，用于存放shell脚本，shell脚本主要用来实现发送钉钉通知。
  2. vim daily_work_remind.sh，写日报提醒脚本，access_token指定为你的钉钉机器人webhook。
  1. vim weekly_work_remind.sh，写周报提醒脚本，access_token指定为你的钉钉机器人webhook。

3、配置定时任务

采用在crontab文件配置的方式，因此直接vim /etc/crontab，新增如下4个定时任务配置。

# 每周一到周五17:55执行日报提醒
55 17 * * 1-5 root /usr/local/work/daily_work_remind.sh

# 每周六16:50执行日报提醒（苦逼，近期公司要求周六正常上班）
50 16 * * 6 root /usr/local/work/daily_work_remind.sh

# 每周六16:55执行本周周报提醒
55 16 * * 6 root /usr/local/work/weekly_work_remind.sh this

# 每周一9:20到9:40之间每隔10分钟执行上周周报提醒，提醒3次
20-40/10 9 * * 1 root /usr/local/work/weekly_work_remind.sh last

大数据开发-kappa架构

发表于 2020-04-19 更新于 2023-06-15 分类于 大数据 Changyan：

Kappa架构

Kappa 架构可以认为是 Lambda 架构的简化版（只要移除 lambda 架构中的批处理部分即可）

批流一体化：

Nginx -> kafka -> StructuredStreaming/Flink -> HDFS/实时数据处理。

然后两种输出模式：

1、FileSink：写到文件体系中（HDFS文件系统），可以设定trigger(Trigger.ProcessingTime(“60 seconds”))，每10分钟生层一个文件，做离线数据。

2、ForeachSink：自定义输出，一般可以输出到mysql，Redis，Hbase。等做实时开发。

在真实的场景中，很多时候并不是完全规范的 Lambda 架构或 Kappa 架构，可以是两者的混合，比如大部分实时指标使用 Kappa 架构完成计算，少量关键指标（比如金额相关）使用 Lambda 架构用批处理重新计算，增加一次校对过程。

业务库数据开发：

业务库的数据进行binlog监听，获取线上业务的数据

Mysql ->Canal ->binlog -> Kafka -> Streaming/Flink

大数据开发-lambda架构

发表于 2020-03-19 更新于 2023-06-15 分类于 大数据 Changyan：

lambda架构

Lambda 架构总共由三层系统组成：批处理层(Batch Layer)，速度处理层(Speed Layer)，以及用于响应查询的服务层(Serving Layer)。

离线开发：

nginx(日志获取) -> logrotate(日志分割并轮转10分钟一次) -> Flume抽取 -> HDFS

缺点：

同样的需求需要开发两套一样的代码。同样的逻辑计算两次，整体资源占用会增多

实时开发：

Nginx -> kafka -> StructuredStreaming/Flink -> ES/Mysql
