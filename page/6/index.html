SSH（安全外壳协议）

SSH 为 [Secure Shell](https://baike.baidu.com/item/Secure Shell) 的缩写，由 IETF 的网络小组（Network Working Group）所制定；SSH 为建立在应用层基础上的安全协议。SSH 是较可靠，专为远程登录会话和其他网络服务提供安全性的协议。利用 SSH 协议可以有效防止远程管理过程中的信息泄露问题。SSH最初是UNIX系统上的一个程序，后来又迅速扩展到其他操作平台。SSH在正确使用时可弥补网络中的漏洞。

SSH应用场景

SSH在Linux环境运用及其广泛，也为开发人员带来很大的遍历，提升安全度。

大数据底座-CDH、HDP、CDP

发表于 2020-06-19 更新于 2023-06-15 分类于 大数据 Changyan：

概述

说到大数据，离不开Hadoop。大数据涉及的组件是在太多了，各个组件存在兼容适配问题。那么就会有大数据底座的概念。

底座主要有：Apache（最原始的版本，所有发行版均基于这个版本进行改进）、Cloudera版本（Cloudera’s Distribution Including Apache Hadoop，简称CDH）、Hortonworks版本(Hortonworks Data Platform，简称HDP），对于国内而言，绝大多数选择CDH版本

Hadoop的发行版本有很多，有华为发行版，Intel发行版，Cloudera发行版（CDH），MapR版本,以及HortonWorks（HDP）、Transwarp Data Hub (TDH)版本等。所有发行版都是基于Apache Hadoop衍生出来的，产生这些版本的原因，是由于Apache Hadoop的开源协议决定的：任何人可以对其进行修改，并作为开源或商业产品发布和销售。

以下主要记录一下CDH、HDP、CDP

CDH是你一个很好地选择。

大数据之难

发表于 2020-06-19 更新于 2023-06-15 分类于 大数据 Changyan：

作为java开发转大数据，个人对大数据之难的感受：

  • 数据之难

1、知识体系庞大，解决方案多导致架构多样化；

2、环境搭建复杂，涉足组件多，版本依赖性严重，占用资源多；

3、实战少，难以提升；

4、需结合实际场景，场景应该来源于运营人员的需求，而不是开发人员自行设定；

  • 大数据组件标准

虽然大数据有很多框架组件，但其实大部分数据库语句都类似，所以并没有想象得难，大量的技术组件百花齐放，但大浪淘沙，终究会尘埃落定。

  • 建议

1、确定需要学习的大数据场景及涉及生态组件，不同场景规划技术框图；

2、准备好自动管理的linux脚本，方便快速恢复开发环境；

3、模拟数据，从数据源到数据处理，最后数据输出，形成开发闭环；

4、熟悉不同架构的组件及API，模拟场景进行运用；

5、平常心，只是接触晚，实践少，并不代表大数据很难；

Scala简介

联邦理工学院洛桑（EPFL）的Martin Odersky于2001年基于Funnel的工作开始设计Scala。

Scala 是一门多范式（multi-paradigm）的编程语言，设计初衷是要集成面向对象编程和函数式编程的各种特性。

Scala 运行在 Java 虚拟机上，并兼容现有的 Java 程序。

Scala 源代码被编译成 Java 字节码，所以它可以运行于 JVM 之上，并可以调用现有的 Java 类库。

Scala 基础

  • class和object区别

object

在scala中没有静态方法和静态字段，所以在scala中可以用object来实现这些功能

scala中object：所有成员变量和方法默认都是 static 的，所以可以直接访问main方法

class和object的一个差别是,单例对象不带参数,而类可以.因为你不能用new关键字实例化一个单例对象,所以没有机会传递给它参数

class

在scala中，类名可以和对象名为同一个名字，该对象称为该类的伴生对象,类和伴生对象可以相互访问他们的私有属性,但是它们必须在同一个源文件中

Scala中的下划线到底有多少种应用场景

1、作为“通配符”，类似Java中的*。如import scala.math._

2、:*作为一个整体，将某个参数当作参数序列处理！例如val s = sum(1 to 10:*)就是将1 to 10当作参数序列处理。

3、指代一个集合中的每个元素。例如我们要在一个Array a中筛出偶数，并乘以2，可以用以下办法：
a.filter(%2==0).map(2*)。
又如要对缓冲数组ArrayBuffer b排序，可以这样：
val bSorted = b.sorted(_

4、在元组中，可以用方法_1, _2, _3访问组员。如a._2。其中句点可以用空格替代。

5、使用模式匹配可以用来获取元组的组员，例如
val (first, second, third) = t
但如果不是所有的部件都需要，那么可以在不需要的部件位置上使用_。比如上一例中val (first, second, _) = t

6、还有一点，下划线_代表的是某一类型的默认值。
对于Int来说，它是0。
对于Double来说，它是0.0
对于引用类型，它是null。
