大数据

所谓大数据，即是大的数据处理，我们得清楚大数据来源，数据永远来源于实际业务场景，引发的大数据缘由如下：

历史数据累积，进一步对其分析处理；
并发数据，抢红包、抢车票、网络攻击等大数据分析；
高业务量场景，滴滴、地图分析、千人千面；
。。。。

综上所述，处理传统业务系统的同学总会觉得接触不到数据，其实大数据同样来源于业务数据。

例如，千人千面系统，也是基于用户实际的业务场景，对用户操作行为进行埋点数据存储，当用户多或日积月累用户行为数据较多了便能对其进行分析处理。

从而形成了大数据的说法。大数据最终也是回归到实际业务场景，例如千人千面推荐系统，根据用户行为以T+1生成用户推荐数据，这些推荐数据交由给业务系统反显给用户。

刚才说到T+1作为离线数据处理，如果是实时计算一般都是针对海量数据进行的，一般要求为秒级。实时计算主要分为两块:数据的实时入库、数据的实时计算。

REmote DIctionary Server(Redis) 是一个由 Salvatore Sanfilippo 写的 key-value 存储系统，是跨平台的非关系型数据库。

Redis 是一个开源的使用 ANSI C 语言编写、遵守 BSD 协议、支持网络、可基于内存、分布式、可选持久性的键值对(Key-Value)存储数据库，并提供多种语言的 API。

##基础数据类型
5大数据类型：string、list、hash、set、zset

3个高级数据结构：Bitmaps、Hyperloglogs、GEO。

Strings：最大512MB

Sorted sets：与Sets类型类似，没有明确说明。应该也是2^32-1个数据
