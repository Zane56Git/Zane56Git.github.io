作为以为IT人员，一定要重视数据、了解数据、分析数据，才能应用数据。

数据级别

首先先了解一下数据级别：

1M = 1024k = 1048576字节

算法是：

1Byte(字节) = 8bit(位)

1KB = 1024Byte(字节)

1MB = 1024KB

1GB = 1024MB

1TB = 1024GB

一个汉字要占用2个字节，如果换算成中文汉字那么就是1M=524288个汉字

从Byte、KB、MB、GB、TB到PB、EB、ZB、YB

数据处理级别

场景：一天1.5亿条消费数据

从kafka到mongo、redis；单节点12000 消费速率

OLAP与OLTP的介绍和区别

数据处理大致可以分成两大类：联机事务处理OLTP（on-line transaction processing）、联机分析处理OLAP（On-Line Analytical Processing）。OLTP是传统的关系型数据库的主要应用，主要是基本的、日常的事务处理，例如银行交易。OLAP是数据仓库系统的主要应用，支持复杂的分析操作，侧重决策支持，并且提供直观易懂的查询结果。

linux生成多对秘钥并指定秘钥登录

Linux通过秘钥远程连接

密钥目录（默认，也可以指定，下面说明） 

cd ~/.ssh/
  1. 秘钥生成命令
    ssh-keygen

执行完上述的命令就会在root目录下生成公钥跟私钥文件

  • /root/.ssh/.id_rsa 私钥
  • /root/.ssh/.id_rsa.pub 公钥

  1. 把公钥copy到要登录的目标机器上

  2. 在目标机器上将id_rsa.pub放到想要登录的用户的主目录的.ssh目录里面，并且重命名为authorized_keys，最重要的是要修改authorized_keys权限为600

  3. 在有私钥的机器上远程链接上面配置了公钥的机器

ssh user_name@ip

  1. 生成秘钥对并指定秘钥对名称（切换值需生成的目录）：

例如切换到.ssh_test目录生成了test_rsa（公钥）、test_rsa.pub（私钥）

ssh-keygen -t rsa -f test_rsa -C “for test”
例如：
ssh-keygen -t rsa -C “123@qq.com“

  • 输入密码（可直接回车）
  • 确认密码（可直接回车）

2.将test.pub公钥添加至要免密登录的服务器

ssh-copy-id -i ~/.ssh/id_rsa.pub root@ip

或

ssh-copy-id -i ~/.ssh/test_rsa.pub root@ip

3.使用指定other私钥登录远程服务器(默认22)：

ssh -i ~/.ssh/test ip -p36000

移动目录，指定私钥(注意密钥目录权限，管理员可读写，其他人无访问权限)

ssh -i /Users/zane/.ssh_test/test_rsa user@ip
