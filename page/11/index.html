Spark SQL *

http://spark.apache.org/sql/

Spark SQL是Spark用来处理结构化数据的一个模块，它提供了一个编程抽象叫做DataFrame并且作为分布式SQL查询引擎的作用。

spark SQL主要用于进行结构化数据的处理。它提供的最核心的编程抽象就是DataFrame。

为什么要学习Spark SQL？

学习了Hive后，我们指定它是将Hive SQL转换成MapReduce然后提交到集群上执行，大大简化了编写MapReduce的程序的复杂性，由于MapReduce这种计算模型执行效率比较慢。所以Spark SQL的应运而生，它是将Spark SQL转换成RDD，然后提交到集群执行，执行效率非常快！同时Spark SQL也支持从Hive中读取数据。

运行原理

将 Spark SQL 转化为 RDD， 然后提交到集群执行

  • 容易整合
  • 统一的数据访问方式
  • 兼容Hive
  • 标准的数据连接

pb级别数据

发表于 2019-03-18 更新于 2023-06-15 分类于 数据结构 Changyan：

pb指petabyte，它是62616964757a686964616fe78988e69d8331333365646234较高级的存储单位，其上还有EB,ZB,YB等单位，1PB=1024TB。

特此一篇记录数据应用

为什么是PB级别，因为大部分业务场景中我们都能接触到GB、TB级别的数据，当然，如果是小厂，可能TB数据都很少接触到。

再者，为什么要去讨论数据级别呢，因为很多时候我们在将业务场景的时候，其实和业务所涉及的数据级别是密不可分的；

我们不能就1年几百M的数据或几G的数据也去疯狂地去比拼大数据（当然大数据并不是单说数据量大），显然是不合理，架构设计涉及到数据规模和数据增长。

如果我们有PB及的数据那得需要多少台电脑（拿普通电脑来说，这样比较清晰），1PB就需要到1024台1T的电脑硬盘了

数据库-NoSQL

发表于 2018-12-06 更新于 2023-06-15 分类于 数据库 Changyan：

数据库主要分为两种类型：SQL和NoSQL。

SQL数据库通常被称为关系型数据库管理系统(RDBMS)。由于此类系统主要利用基于行的数据库结构，连接各个数据表之间的相关数据对象，

而NoSQL数据库并没有任何固定用于保存数据的结构化数据表。从技术上讲，所有非关系型数据库都可以被称为NoSQL数据库。不同于关系型数据库，NoSQL数据库不但可以被快速地设置，并且只需最少量的预先规划(pre-planning)。

常见的NoSQL数据库示例包括：Redis、MongoDB、DynamoDB、SimpleDB、CouchDB、CouchBase、OrientDB、InfiniteGraph、Neo4j、FlockDB、Cassandra、以及HBase等。

实际连接数分析

单机redis启动后，连接数为0，如果通过可视化工具打开连接到redis，则连接数为0；

如果再打开命令行，则连接数+1，为2；（每打开一次命令行连接，就多一个连接）

空闲连接数是指应用端占用空闲连接数

最大连接数是指应用端所限制的最大连接数，并非redis服务端的连接数

当应用最小连接数为2，操作时连接数最大为2

集群redis启动后，一个节点 的连接数为集群主节点数-1，例如三主三从，1个主节点连接数为2
