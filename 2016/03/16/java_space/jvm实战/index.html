查看GC日志

jar启动命令（输出GC日志）
java -jar .\app.jar -verbose:gc -XX:+PrintGCDetails -XX:+PrintHeapAtGC -Xloggc:gc-app.log

例如，查询虚拟机的运行状态信息，每隔1000ms，总共5次。
jstat -gc 5524 1000 5

导出Dump文件#

使用jstack精确找到异常代码

1、找到CPU利用率持续比较高的进程，获取进程号，此处PID为3036
命令：
top

2、找到上述进程中，CPU利用率比较高的线程号TID（十进制数）,此处为3046
命令：
ps p 5262 -L -o pcpu,pid,tid,time,tname,cmd

3、将获取的线程号（十进制数）转换成十六进制,此处为0xb46

命令：
printf “%x\n” 5433

查看进程PID为5433中

4、nid为0xb46的线程信息。
命令：
jstack -l 3036

================jvm cpu过高排查实战============9205、top -Hp 9798

1、top找出cpu高的java进程号9592

2、top -Hp 9592查看cpu占用time最高的线程编号28178

3、执行 printf “%x\n” 28178 获取线程十六进制地址6e12 （十六进制一定要小写）

4、执行 jstack 9592（pid）|grep -10 6e12（线程十六进制号）

jstack 9205（pid）|grep -10 28fa
注意:需要切换到运行时的用户环境
jstack 2576 | grep -A 10 a10

dump分析

  • jps

JVM Process Status Tool,显示指定系统内所有的HotSpot虚拟机进程。

  • jstat

jstat(JVM statistics Monitoring)是用于监视虚拟机运行时状态信息的命令，它可以显示出虚拟机进程中的类装载、内存、垃圾收集、JIT编译(Just In Time Compiler, 即时编译器)等运行数据。

  • jhat

jhat(JVM Heap Analysis Tool)命令是与jmap搭配使用，用来分析jmap生成的dump，jhat内置了一个微型的HTTP/HTML服务器，生成dump的分析结果后，可以在浏览器中查看。在此要注意，一般不会直接在服务器上进行分析，因为jhat是一个耗时并且耗费硬件资源的过程，一般把服务器生成的dump文件复制到本地或其他机器上进行分析。
PS C:\Program Files\Java\jdk1.8.0_40\bin> .\jhat.exe -J-Xmx512m C:\Users\Admin\Desktop\4098xxx.dump
http://localhost:7000/
